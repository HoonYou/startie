## 발표 한 줄 주제

- 스타티의 무료체험 유저를 더 많이 유료로 전환시키기 위한 제품·퍼널·채널 최적화 전략

## 발표 핵심 스토리

1. 유료전환은 모든 유저에게 동일하게 일어나지 않는다.
2. 초기 가치 경험(Aha)에 도달한 유저가 훨씬 잘 전환된다.
3. 전환 직전 병목은 `Pricing Viewed -> Checkout Started`, `Checkout Started -> Payment Completed` 구간이다.
4. 따라서 다음 분기 우선순위는 `온보딩 강화`, `체크아웃 개선`, `실패 결제 회복`, `저효율 채널 축소`다.

## 발표 구조

### 1. 문제 정의
- 질문 정의:
  - 현재 스타티의 유료전환은 어디서 막히고 있는가?
- 핵심 지표:
  - 체험판 대비 유료전환율, 단계별 퍼널 전환율
- 사용 테이블:
  - `event_logs`, `payment_transactions`, `users`
- 의사결정 기준(So What):
  - 최대 병목 구간이 확인되면 해당 단계 개선이 최우선이다.

### 2. 가치 경험과 전환
- 질문 정의:
  - 어떤 초기 행동이 유료전환을 끌어올리는가?
- 핵심 지표:
  - Activation rate, Aha rate, Aha vs Non-Aha 전환율
- 사용 테이블:
  - `users`, `event_logs`, `payment_transactions`
- 의사결정 기준(So What):
  - Aha 행동이 강하면 온보딩 KPI를 Aha 중심으로 재정의한다.

### 3. 병목과 이탈
- 질문 정의:
  - 유저는 왜 결제까지 가지 못하는가?
- 핵심 지표:
  - 가격 페이지 이탈률, 체크아웃 이탈률, 결제 실패율, 실패 후 churn
- 사용 테이블:
  - `event_logs`, `payment_transactions`, `chat_events`, `users`
- 의사결정 기준(So What):
  - 결제 실패 복구가 효과적이면 CS/CRM 자동화에 투자한다.

### 4. 실험과 채널
- 질문 정의:
  - 어떤 실험과 어떤 채널이 실제로 전환 최적화에 기여하는가?
- 핵심 지표:
  - 온보딩 uplift, 채널별 최종 전환율, LTV/CAC
- 사용 테이블:
  - `experiments`, `ab_assignment`, `campaigns`, `users`, `payment_transactions`
- 의사결정 기준(So What):
  - 실험은 확대 여부를 판단하고, 채널은 예산 재배분으로 연결한다.

### 5. 다음 분기 제안
- 질문 정의:
  - 무엇을 먼저 해야 가장 빠르게 유료전환을 올릴 수 있는가?
- 핵심 지표:
  - 예상 전환율 개선 폭, 예상 MRR 임팩트, 실행 난이도
- 사용 테이블:
  - Q1~Q4 종합
- 의사결정 기준(So What):
  - 영향도와 실행 속도가 높은 전략부터 실행한다.

## CEO용 최종 제안

### 제안 1. 온보딩 KPI를 Aha 도달로 재설계
- What:
  - 단순 가입보다 `강의 완료 + 퀴즈 제출` 도달 여부가 더 중요하다.
- Why:
  - 초기 가치 체감이 유료전환 가능성을 높인다.
- So What:
  - 7일 내 Aha 도달을 온보딩 핵심 목표로 둔다.
- Impact:
  - 초기 전환율과 D7 리텐션 동시 개선 기대

### 제안 2. 체크아웃 병목 해소
- What:
  - 가격 페이지 이후와 결제 시작 이후 이탈이 크다.
- Why:
  - 가격 설득력 부족 또는 결제 UX 마찰 가능성이 높다.
- So What:
  - 가격 페이지 메시지 개선, 결제 단계 축소, 실패 사유별 복구 플로우를 만든다.
- Impact:
  - 직접적인 유료전환율 상승 기대

### 제안 3. 실패 결제 복구 자동화
- What:
  - 실패 결제 이후 방치된 유저가 이탈한다.
- Why:
  - 결제 문제 해결 장치가 약하다.
- So What:
  - billing chat, push, 리마인더를 묶은 회복 시나리오를 자동화한다.
- Impact:
  - 회수 가능한 유료전환 추가 확보

### 제안 4. 저효율 채널 예산 축소
- What:
  - 일부 채널은 유입은 만들지만 유료전환 효율이 낮다.
- Why:
  - 타겟-메시지-랜딩의 정합성이 떨어질 수 있다.
- So What:
  - 저효율 paid 채널 예산을 전환 효율이 높은 채널로 재배분한다.
- Impact:
  - 같은 예산으로 더 높은 전환과 더 낮은 CAC 기대
